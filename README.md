# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

### Интерфейсы и типы

#### Интерфейс `IProduct`
Описывает товар из каталога магазина.

Поля:
- `id: string` — уникальный идентификатор товара
- `description: string` — описание товара
- `image: string` — путь к изображению товара
- `title: string` — название товара
- `category: string` — категория товара
- `price: number | null` — цена товара; null означает, что товар недоступен для покупки

#### Интерфейс `IBuyer`
Описывает данные покупателя, которые тот вводит при оформлении заказа.

Поля:
- `payment: TPayment` — выбранный способ оплаты
- `email: string` — email покупателя
- `phone: string` — телефон покупателя
- `address: string` — адрес доставки

#### Тип `TPayment`
Описывает доступные способы оплаты: `'card' | 'cash'`

#### Интерфейс `IProductsResponse`
Описывает объект, который возвращает сервер при запросе каталога товаров.

Поля:
- `total: number` — общее количество товаров
- `items: IProduct[]` — массив товаров

#### Интерфейс `IOrderRequest`
Описывает данные, отправляемые на сервер при оформлении заказа. Расширяет `IBuyer`, добавляя список товаров и сумму.

Поля:
- `items: string[]` — массив id выбранных товаров
- `total: number` — итоговая сумма заказа

#### Интерфейс `IOrderResponse`
Описывает ответ сервера после успешного оформления заказа.

Поля:
- `id: string` — идентификатор созданного заказа
- `total: number` — сумма заказа

## Модели данных

### Класс `ProductsModel`
Отвечает за хранение каталога товаров и товара, выбранного для подробного просмотра.

Конструктор не принимает параметров.

Поля класса:
- `private _items: IProduct[]` — массив всех товаров каталога
- `private _preview: IProduct | null` — товар, открытый для подробного просмотра

Методы класса:
- `setItems(items: IProduct[]): void` — сохраняет переданный массив товаров в модель
- `getItems(): IProduct[]` — возвращает массив всех товаров каталога
- `getItemById(id: string): IProduct | undefined` — возвращает товар по его id или undefined, если товар не найден
- `setPreview(item: IProduct): void` — сохраняет товар для подробного просмотра
- `getPreview(): IProduct | null` — возвращает товар, выбранный для подробного просмотра

---

### Класс `BasketModel`
Отвечает за хранение товаров, которые пользователь добавил в корзину, и операции над ними.

Конструктор не принимает параметров.

Поля класса:
- `private _items: IProduct[]` — массив товаров в корзине

Методы класса:
- `getItems(): IProduct[]` — возвращает текущий список товаров в корзине
- `addItem(item: IProduct): void` — добавляет товар в корзину
- `removeItem(id: string): void` — удаляет товар из корзины по его id
- `clear(): void` — полностью очищает корзину
- `getTotalPrice(): number` — возвращает суммарную стоимость товаров в корзине; товары без цены (null) считаются как 0
- `getCount(): number` — возвращает количество товаров в корзине
- `hasItem(id: string): boolean` — проверяет наличие товара в корзине по id; возвращает true или false

---

### Класс `BuyerModel`
Отвечает за хранение, обновление и валидацию данных покупателя при оформлении заказа.

Конструктор не принимает параметров.

Поля класса:
- `private _payment: TPayment | null` — выбранный способ оплаты
- `private _email: string` — email покупателя
- `private _phone: string` — телефон покупателя
- `private _address: string` — адрес доставки

Методы класса:
- `setField(field: keyof IBuyer, value: string): void` — обновляет значение одного поля по его имени, не затрагивая остальные поля
- `getData(): IBuyer` — возвращает объект со всеми текущими данными покупателя
- `clear(): void` — сбрасывает все поля в исходное пустое состояние
- `validate(): Partial<Record<keyof IBuyer, string>>` — проверяет заполненность всех полей и возвращает объект с ошибками. Если поле заполнено — оно отсутствует в объекте. Если пустое — содержит текст ошибки. Например: `{ payment: 'Не выбран способ оплаты', email: 'Введите email' }`

## Слой коммуникации

### Класс `LarekApi`
Отвечает за взаимодействие приложения с сервером «Веб-ларёк». Использует объект класса `Api` через композицию — принимает его в конструктор и вызывает его методы `get` и `post`.

Конструктор:
`constructor(api: IApi)` — принимает объект, реализующий интерфейс `IApi`

Поля класса:
- `private _api: IApi` — объект для выполнения HTTP-запросов к серверу

Методы класса:
- `getProducts(): Promise<IProductsResponse>` — выполняет GET запрос на эндпоинт `/product/` и возвращает объект с массивом товаров
- `createOrder(order: IOrderRequest): Promise<IOrderResponse>` — выполняет POST запрос на эндпоинт `/order/` с данными заказа и возвращает подтверждение покупки с id заказа и суммой

## Слой представления (View)

Все классы представления наследуют базовый класс `Component`. Они не хранят данные и не принимают решений — только отображают то, что им передают снаружи, и сообщают о действиях пользователя через брокер событий.

### Класс `Page`
Отвечает за главную страницу: галерею товаров и счётчик корзины в шапке.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает корневой элемент страницы и брокер событий.

Поля класса:
- `private _catalog: HTMLElement` — контейнер галереи товаров
- `private _counter: HTMLElement` — элемент счётчика на иконке корзины
- `private _basketButton: HTMLElement` — кнопка корзины в шапке

Сеттеры:
- `set catalog(cards: HTMLElement[])` — принимает массив DOM-элементов карточек и вставляет их в галерею
- `set counter(value: number)` — обновляет число на иконке корзины

При клике на кнопку корзины генерирует событие `basket:open`.

---

### Класс `Modal`
Управляет модальным окном. Не имеет дочерних классов — любой контент передаётся снаружи через сеттер. Закрывается по клику на крестик или по клику на оверлей вне окна.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)` — принимает элемент модального окна и брокер событий.

Поля класса:
- `private _content: HTMLElement` — контейнер для контента внутри модального окна
- `private _closeButton: HTMLElement` — кнопка закрытия (крестик)

Сеттеры:
- `set content(value: HTMLElement)` — вставляет переданный элемент внутрь модального окна

Методы класса:
- `open(): void` — добавляет модификатор `modal_active`, делая окно видимым
- `close(): void` — убирает модификатор `modal_active` и очищает контент

---

### Класс `Card` (абстрактный)
Родительский класс для всех вариантов карточки товара. Содержит общие поля и сеттеры, которые используются во всех дочерних классах.

Конструктор:
`constructor(container: HTMLElement)` — принимает DOM-элемент карточки.

Поля класса:
- `protected _title: HTMLElement`
- `protected _price: HTMLElement`
- `protected _category: HTMLElement | null`
- `protected _image: HTMLImageElement | null`

Сеттеры:
- `set title(value: string)`
- `set price(value: number | null)` — если цена null, выводит «Бесценно»
- `set category(value: string)` — устанавливает текст и CSS-модификатор из `categoryMap`
- `set image(src: string)`

#### Класс `CardCatalog`
Карточка товара в галерее на главной странице. При клике генерирует событие `card:select` с id товара.

#### Класс `CardPreview`
Детальная карточка товара в модальном окне. Содержит описание и кнопку «Купить» / «Удалить из корзины». Если цена товара равна null — кнопка блокируется с текстом «Недоступно».

Дополнительные поля:
- `private _button: HTMLButtonElement`
- `private _description: HTMLElement`

Дополнительные сеттеры:
- `set description(value: string)`
- `set inBasket(value: boolean)` — переключает текст кнопки
- `set price(value: number | null)` — переопределяет родительский сеттер, дополнительно блокирует кнопку если цена null

При клике на кнопку генерирует событие `card:buy` с id товара и флагом `inBasket`.

#### Класс `CardBasket`
Карточка товара внутри корзины. Показывает порядковый номер, название и цену.

Дополнительные поля:
- `private _index: HTMLElement`
- `private _deleteButton: HTMLButtonElement`

Дополнительные сеттеры:
- `set index(value: number)` — порядковый номер в списке

При клике на кнопку удаления генерирует событие `basket:remove` с id товара.

---

### Класс `Basket`
Компонент корзины. Показывает список товаров, итоговую сумму и кнопку оформления заказа.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля класса:
- `private _list: HTMLElement` — контейнер списка товаров
- `private _total: HTMLElement` — элемент с итоговой суммой
- `private _orderButton: HTMLButtonElement` — кнопка «Оформить»

Сеттеры:
- `set items(cards: HTMLElement[])` — вставляет карточки в список; если массив пустой — показывает текст «Корзина пуста» и блокирует кнопку оформления
- `set total(value: number)` — обновляет итоговую сумму

При клике на «Оформить» генерирует событие `order:open`.

---

### Класс `Form` (абстрактный)
Родительский класс для форм оформления заказа. Обрабатывает ввод данных и сабмит формы.

Конструктор:
`constructor(container: HTMLFormElement, events: IEvents)`

Поля класса:
- `protected _submitButton: HTMLButtonElement`
- `protected _errors: HTMLElement`
- `protected _form: HTMLFormElement`

Сеттеры:
- `set valid(value: boolean)` — включает или отключает кнопку submit
- `set errors(value: string)` — показывает текст ошибки под формой

При изменении любого инпута генерирует событие `form:change` с именем поля и значением. При сабмите генерирует событие `<имя_формы>:submit`.

#### Класс `OrderForm`
Форма первого шага оформления: выбор способа оплаты и адрес доставки.

Дополнительные поля:
- `private _paymentButtons: HTMLButtonElement[]` — кнопки выбора способа оплаты

Дополнительные сеттеры:
- `set payment(value: string)` — выделяет активную кнопку модификатором `button_alt-active`

При клике на кнопку оплаты генерирует событие `payment:select` с выбранным способом.

#### Класс `ContactsForm`
Форма второго шага оформления: ввод email и телефона. Не добавляет логики сверх родительского класса.

---

### Класс `Success`
Экран успешного оформления заказа. Показывает сумму списанных синапсов и кнопку возврата к покупкам.

Конструктор:
`constructor(container: HTMLElement, events: IEvents)`

Поля класса:
- `private _description: HTMLElement` — элемент с текстом о списанной сумме
- `private _closeButton: HTMLElement` — кнопка «За новыми покупками!»

Сеттеры:
- `set total(value: number)` — обновляет текст с суммой

При клике на кнопку генерирует событие `success:close`.

## События

### События от моделей данных

`catalog:changed` — генерирует `ProductsModel` после сохранения массива товаров. Сигнализирует презентеру, что нужно перерисовать каталог.

`preview:changed` — генерирует `ProductsModel` после сохранения товара для просмотра. Презентер открывает модальное окно с детальной карточкой.

`basket:changed` — генерирует `BasketModel` при любом изменении корзины: добавлении, удалении или очистке. Презентер обновляет счётчик в шапке.

`buyer:changed` — генерирует `BuyerModel` при изменении любого поля покупателя. Презентер валидирует форму и обновляет состояние кнопки и текст ошибок.

### События от представлений

`card:select` — генерирует `CardCatalog` при клике на карточку в каталоге. Передаёт id товара.

`card:buy` — генерирует `CardPreview` при нажатии кнопки в детальной карточке. Передаёт id товара и флаг `inBasket`, по которому презентер решает добавить или удалить товар.

`basket:open` — генерирует `Page` при клике на иконку корзины в шапке.

`basket:remove` — генерирует `CardBasket` при клике на кнопку удаления внутри корзины. Передаёт id товара.

`order:open` — генерирует `Basket` при нажатии кнопки «Оформить».

`payment:select` — генерирует `OrderForm` при выборе способа оплаты. Передаёт значение `card` или `cash`.

`form:change` — генерирует базовый класс `Form` при вводе данных в любое поле. Передаёт имя поля и его текущее значение.

`order:submit` — генерирует `OrderForm` при отправке первой формы. Презентер открывает вторую форму.

`contacts:submit` — генерирует `ContactsForm` при отправке второй формы. Презентер отправляет заказ на сервер.

`success:close` — генерирует `Success` при клике на кнопку «За новыми покупками!». Презентер закрывает модальное окно.

## Презентер

Роль презентера выполняет основной скрипт приложения — `src/main.ts`. Выносить его в отдельный класс не потребовалось: приложение одностраничное, логика линейная, и держать всё в одном файле проще для понимания.

Презентер только обрабатывает события — не генерирует их. Каждый обработчик подписывается на одно событие и выполняет ровно одно действие: либо обновляет данные в модели, либо перерисовывает нужный компонент представления.

Представление перерисовывается в двух случаях:
- при обработке события от модели данных (`catalog:changed`, `preview:changed`, `basket:changed`, `buyer:changed`)
- при открытии модального окна

### Порядок инициализации

1. Создаётся брокер событий `EventEmitter`
2. Создаются модели данных, каждой передаётся брокер
3. Создаются компоненты `Page` и `Modal`, привязанные к DOM
4. Регистрируются все обработчики событий
5. Выполняется запрос к серверу за товарами; после ответа вызывается `productsModel.setItems()`, что запускает событие `catalog:changed` и отрисовывает каталог
